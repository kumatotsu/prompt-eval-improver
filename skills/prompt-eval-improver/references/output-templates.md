# Output Templates

`prompt-eval-improver` の出力を揃えるためのテンプレートです。タスクの重さに応じて簡易版 / 詳細版を使い分けてください。

## 簡易版（標準）

以下は、軽量な改善依頼向けの最小構成です。採点表は不要な限り出しません。

````markdown
## 前提・仮定
- ...

## 問題点（優先度順）
1. ...
2. ...

## 改善後プロンプト
```text
...
```

## 実行環境メモ
- ChatGPT / API / Codex のどれ向けか:
- 追加で分離すべき設定:

## 変更理由（要点）
- ...

## 確認手順
- 正常系:
- 失敗系:

## 確認したい点（必要時のみ）
- ...
````

## 詳細版（実務レビュー）

以下は、評価レポートと差分説明まで必要な場合の構成です。`10項目×10点` が必要なときだけ使います。

````markdown
## 前提・仮定
- 対象モデル:
- タスク:
- 成功条件:
- 未確定点:

## 現行プロンプトの問題点
| 観点 | 問題 | 影響 | 優先度 |
|---|---|---|---|
| 目的適合性 | ... | ... | 高 |

## 改善後プロンプト（コピペ可能）
```text
...
```

## 実行環境別の実装メモ
- ChatGPT:
- API:
- Codex / agentic:

## API向け prompt object 案（必要時のみ）
```yaml
version: ...
variables:
  ...
messages:
  - role: system
    content: ...
  - role: user
    content: ...
```

## 差分要約
| 変更箇所 | 改善前 | 改善後 | 意図 |
|---|---|---|---|
| 出力仕様 | ... | ... | ... |

## 評価レポート（観点別）
| 観点 | 評価 | 根拠 | 改善余地 |
|---|---|---|---|
| 入出力契約 | 8/10 | ... | ... |

## テスト観点 / 失敗ケース
- 正常系:
- 境界系:
- 失敗系:

## 最小 eval 案（必要時のみ）
- ケース:
- grader 観点:
- 合格条件:

## 追加質問（必要時のみ）
- ...
````

## 高リスク用途向けの追記候補

- 禁止事項（法務 / 規約 / 個人情報）
- 出力拒否条件
- 情報不足時の応答テンプレート
- 監査ログ向けの根拠記載ルール（必要な場合のみ）
